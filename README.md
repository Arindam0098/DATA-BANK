1. Project Overview
This project contains SQL solutions for the Data Bank Case Study, focusing on customer allocation, regional performance, 
and transaction analysis. The analysis explores customer distribution across network nodes, transaction behavior, regional transaction volumes, 
customer migration between nodes, and monthly banking activity using SQL.

2. Objectives
Analyze customer allocation across regions and nodes.
Measure regional network performance.
Track customer movement between nodes.
Evaluate deposits, withdrawals, and purchase transactions.
Generate business insights from banking data.

3. SQL Concepts Used
Joins
Aggregate Functions
Window Functions
Common Table Expressions (CTEs)
Create View
Subqueries
Date Functions
CASE Statements

4. Key Business Questions
How many nodes exist in each region?
How are customers distributed across regions?
What is the average reallocation period for customers?
Which regions generate the highest transaction volumes?
How do customer transaction patterns vary over time?
